security:
- backend/main.py: удалён атрибут "onerror" из bleach whitelist для <img>
  (позволял XSS через <img onerror="..."> в тексте гайда)

backend:
- backend/main.py: race condition в кэше категорий — asyncio.Lock
  с double-check pattern после захвата лока
- backend/main.py: валидация key ^[a-z0-9_-]{1,64}$ для категорий и
  гайдов; _validate_key() вызывается в обоих admin upsert endpoints
- backend/main.py: CategoryIn / GuideIn — field_validator для непустого title
- backend/database.py: import_guides — pre-validate FK до транзакции,
  понятная ошибка вместо PG FK violation
- backend/main.py: ValueError из import_guides → HTTP 422

frontend:
- frontend/src/App.jsx: BackButton.offClick перед каждым onClick —
  устраняет накопление обработчиков и N-кратное срабатывание

ОПТИМИЗАЦИИ (performance)
════════════════════════════════════════════════════════════════

- backend/main.py: N+1 для [[key]] ссылок → resolve_guide_links_bulk()
  (один SELECT WHERE key = ANY($1::text[]))
- backend/main.py: format_guide_text → asyncio.to_thread()
- backend/database.py: reorder_guides / reorder_categories →
  один UNNEST batch UPDATE вместо N отдельных UPDATE
- backend/main.py: LRU-кэш гайдов (200 записей, TTL 2 мин),
  инвалидация при upsert/delete
- backend/database.py: get_guides_by_category — теги батч-запросом
  для всей категории за один SELECT

НОВЫЕ ВОЗМОЖНОСТИ (features)
════════════════════════════════════════════════════════════════

теги:
- backend: таблица guide_tags, CRUD (get/set/list/by_tag)
- backend: GET /api/tags, GET /api/tag/{tag},
  PUT /api/admin/guide/{key}/tags
- frontend: TagBadge / TagsList / TagEditor (Enter или запятая)
- frontend: вкладка "Теги" в GuideEditor, сохраняется вместе с гайдом
- frontend: теги в GuideView (клик → фильтрация), в карточках GuidesView
- frontend: TagResultsView — список гайдов по тегу

счётчик просмотров:
- backend: guides.views BIGINT; increment_views(), get_top_guides()
- backend: POST /api/guide/{key}/view (rate 60/min), GET /api/top
- frontend: ViewsCounter в заголовке гайда (1k-сокращение)
- frontend: views в карточках GuidesView; apiRecordView один раз/mount

комментарии:
- backend: таблица guide_comments; get/add/delete
- backend: GET/POST /api/guide/{key}/comments,
  DELETE /api/guide/{key}/comments/{id}
- backend: валидация (1–1000 символов, bleach strip), rate 10/min
- frontend: CommentsSection — коллапсируемый раздел, относительное
  время, Enter для отправки, удаление своих комментариев

подписки и уведомления:
- backend: таблица user_subscriptions; subscribe/unsubscribe/subscribers
- backend: GET/POST/DELETE /api/subscriptions/{category_key}
- backend: POST /api/internal/notify — возвращает user_ids подписчиков
- bot: notify_subscribers() — рассылка участникам при новом гайде
- frontend: useSubscriptions (оптимистичное обновление + rollback)
- frontend: кнопка "Подписаться/Подписан" в GuidesView (колокольчик)

кнопка "Поделиться":
- frontend: ShareButton в GuideView — tg.switchInlineQuery →
  clipboard deep link → navigator.share (fallback-цепочка)

live preview в редакторе:
- backend: POST /api/guide/__preview__ — server-side рендер через
  format_guide_text (полный HTML с иконками и киберссылками)
- frontend: RichEditor.Превью — debounced запрос 400ms к __preview__,
  спиннер, fallback на локальный renderMd

история изменений (UI):
- frontend: вкладка "📋 История" в AdminView — выбор гайда,
  лог с action badge (Создан/Изменён/Удалён/Импорт), датой,
  user_id, раскрывающимся JSON-снапшотом

оффлайн / Service Worker:
- frontend/public/sw.js: network-first для API (guide/categories),
  cache-first для статики; автоочистка старых кэшей
- frontend/src/main.jsx: регистрация SW
